2.context_processor:上下文处理器。
(1)新建一个index.html和edit.html
在标签内分别添加
(2).在主文件中的主函数处渲染一下模板:
@app.route(‘/‘)
def hello_world():
print ‘index’
return render_template(‘index.html’, username=’zhiliao’)
3.在主文件的edit函数处渲染模板:
@app.route(‘/edit/‘)
return render_template(‘edit.html’,username = ‘zhiliao’ )
4.在before_request函数处也要渲染:
@app.before_request
def my_before_request():
if session.get(‘username’):
g.username = session.get(‘username’, username=’zhiliao’)
5.运行，保存，在主文件和访问地址为edit文件都显示:zhiliao


6.主文件的主函数、edit函数、before_request处都需要一个username=‘zhiliao’，但当网站比较大，视图函数多，时，不符合‘不重复’原则，context_processor可解决该问题。
7.写法与before_request处理器相似，
@app.context_processor
def my_context_processor():
return {‘username’:username} #只要在context_processor函数中返回了该字典，那么在所有模板中都会把该key当做变量来渲染。
8.把主文件的主函数、edit函数中的username=‘zhiliao’删除，保存，运行，主页面和edit页面都会显示:zhiliao


9.总结:
上下文处理器应该返回一个字典，字典中的‘key’会被模板中当成变量来渲染。
上下文处理器中返回的字典，在所有页面中都是可用的。
